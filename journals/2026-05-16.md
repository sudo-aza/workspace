# Journal — 2026-05-16

## 03:00 — tts-retry-2: Bark (zoe was right again)

**Context:** Previous run claimed Bark had a "weights_only compat issue" and disk problems. Zoe called out the excuses as lazy. Time to actually try it.

**Real errors encountered and fixed:**

1. **PyTorch 2.6+ weights_only=True** — This one was real. Bark uses old pickle format that isn't compatible with PyTorch's new default. Fix: monkey-patch `torch.load` to add `weights_only=False` before importing bark.

2. **Disk space with full venv** — First attempt let pip install all bark deps including PyTorch (5.1 GB venv). With 3 model files totaling ~4.4 GB, that's 9.5 GB — over the 10 GB limit. Fix: install `suno-bark --no-deps` then add deps individually (scipy, tokenizers, tqdm, transformers, encodec, soundfile, funcy). Result: 1.3 GB venv.

3. **SUNO_USE_SMALL_MODELS not taking effect** — Env var must be set before bark import, and bark reads it at module level. Verified with `USE_SMALL_MODELS = True` print.

4. **No `language` parameter** — Bark auto-detects language from text. No explicit language kwarg.

**Results:**
- 11.9s German audio in 95.8s = **0.12x realtime** (extremely slow — ~8x slower than XTTS v2)
- 3 model downloads: text_small (1.1 GB) + coarse_small (1.1 GB) + fine_small (1.1 GB) + codec (89 MB)
- Total footprint: 1.3 GB venv + 4.4 GB models = 5.7 GB
- License: MIT (fully open source, commercial OK)
- Quality: expressive but can hallucinate (unprompted sounds/words)
- 13-second generation limit per clip

**Updated engine rankings (4 engines tested, real data):**
1. Piper: 2.9x realtime, 8 voices, 74-109 MB, GPL-3.0
2. Kokoro: 2.9x realtime, 1 voice, 311 MB, permissive
3. XTTS v2: 0.33-0.81x realtime, 65 speakers (incl. female), 3.5 GB, CPML non-commercial
4. Bark: 0.12x realtime, expressive, 5.7 GB, MIT

**Lesson:** The "weights_only" issue was real but trivially fixable with a 3-line monkey-patch. The disk issue was caused by lazy pip dependency resolution, not actual space constraints. With careful management, Bark fits on 10 GB VM with 3.4 GB to spare.
