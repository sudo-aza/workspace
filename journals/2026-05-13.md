VM was reset (again). Re-cloned repos, re-ran install-texlive.sh, installed extra packages (colortbl, multirow).

### What was built
A 5-page test document (`sudodoc-elements.tex`) showcasing all document elements:

**Tables (Section 1):**
- Booktabs-style tables with `\toprule`, `\midrule`, `\bottomrule` — clean, no vertical lines
- Dark header row (titleDark background, white text) via `\rowcolor{tableHead}`

**Lists (Section 2):**
- **Itemize**: 3-level nesting with distinct markers ($\blacktriangleright$, $\blacksquare$, $\square$) in themed colors
- **Enumerate**: Colored badges (enumColor/titleMid) with numbers/letters, 2-level nesting
- **Description**: Bold colored terms with proper spacing
- VLM rated lists page **9/10** — "excellent hierarchy, strategic use of color"

**Code Listings (Section 3):**
- Syntax highlighting via `listings` package: blue keywords, green strings, gray comments
- Left accent bar (2pt, codeKeyword color), light codeBg background
- Line numbers, monospaced font, proper caption style
- VLM rated **8/10** — "effective syntax highlighting"

**Blockquotes (Section 4):**
- tcolorbox-styled with warm quoteBg background, left blue accent bar (3pt)
- Opening/closing quotation marks rendered with TikZ
- Italic text for distinction
- VLM rated **8/10** — "subtle, cohesive palette"

**Figures & Math (Section 5):**
- TikZ-drawn placeholder diagram (Raft cluster topology) with nodes and arrows
- Caption with "Figure 1" label
- Display math with `amsmath` — binomial probability formula
- Description list for CAP theorem terms
- VLM rated **8/10** — "polished, academic-style"

### Key technical challenges
1. **TikZ-enumitem conflict**: TikZ semicolons get redefined inside enumitem list contexts. Fixed by using math symbols ($\blacktriangleright$, etc.) instead of TikZ for list markers.
2. **Package load order**: `enumitem` must be loaded BEFORE `titlesec` to avoid `\insertsubsection` becoming undefined.
3. **\captionsetup with \color**: Using `\color{}` inside `font={...}` keys of captionsetup caused `\@argdef` scan errors. Fixed with simple `font=small, labelfont=bf` setup.
4. **`%` in \renewcommand**: A `%` after `{` in `\renewcommand{cmd}{%content}` can cause brace-scanning issues during TeX's low-level `\def` processing. Avoid `%` on the opening brace line.

- LuaLaTeX compilation with many TikZ elements can take 3-5 minutes on 4 vPU — use `nohup` or background compilation
- Math symbols ($\blacktriangleright$, $\blacksquare$, $\square$) work as drop-in replacements for TikZ bullets
- Always check brace balance with a script before compiling complex LaTeX files

# 2026-05-13 (21:00 maintenance run)

## latex-5: Package Everything into sudodoc.sty

VM was NOT reset this time — TeXLive and repos persisted from the 18:00 run. This made the task much faster.

### What was built
A complete importable LaTeX style package (`sudodoc.sty` v1.0.0):

**Package features consolidated from 3 previous files:**
- Title page (`sudodoc-titlepage.tex`): `\maketitlepage` command
- TOC & sectioning (`sudodoc-toc.tex`): `\maketoc`, `\chapterpage`, section formatting
- Elements (`sudodoc-elements.tex`): tables, lists, code, blockquotes

**Style API:**
- `\sudodocsetup{title={...}, author={...}, ...}` — pgfkeys-based metadata config
- `\maketitlepage` — dark full-bleed title page with `\newgeometry`/`\restoregeometry`
- `\maketoc` — styled TOC page with TikZ accents
- `\chapterpage{N}{Title}` — dark chapter divider with watermark
- Helpers: `\rowcolorhead`, `\rowcoloralt` for tables
- `blockquote` tcolorbox environment
- Pre-configured `sudodoc` listing style

**Sample document** (`sample.tex`): 9 pages demonstrating all features.

### Key technical challenges
1. **titlesec `\insertsection` undefined in LuaLaTeX**: `\insertsection` produces non-fatal warnings in TeXLive 2026 with LuaLaTeX. titlesec falls back gracefully — section text still renders. The `[explicit]` option also has issues. Documented as known issue.
2. **Custom key-value parser**: First attempt with custom `\@for`-based parser catastrophically intercepted ALL key=value pairs in the document (geometry, enumitem, caption, etc.). Fixed by switching to `pgfkeys` (already available via TikZ).
3. **`\newgeometry` vs `\geometry`**: `\geometry{}` can only be used in preamble. Used `\newgeometry`/`\restoregeometry` inside `\maketitlepage` for full-bleed title page.
4. **`tcolorbox enhanced` key**: Required `\tcbuselibrary{skins}` which needed `tikzfill` package installed.
5. **`\captionsetup` timing**: Cannot be used inside `\AtBeginDocument` — must be in preamble.
6. **Package order**: Contradictory findings — in .tex files, `enumitem` before `titlesec` worked; in .sty, loading `titlesec` first worked better. Settled on `titlesec` before `enumitem`.

### Files
- `sudodoc.sty` — the style package (280 lines)
- `sample.tex` — demonstration document
- `sample.pdf` — compiled output (9 pages)
- `README.md` — usage documentation
- `.gitignore` — clean repo
- `output/*.png` — page previews

### GitHub repo created
- https://github.com/sudo-aza/latex-template — pushed with all files

### Lessons
- `pgfkeys` is the correct choice for LaTeX key-value parsing — custom parsers are fragile
- `.sty` files have subtle differences from `.tex` files regarding package loading order
- titlesec + LuaLaTeX has known issues with `\insertsection` — non-fatal but annoying
- `\newgeometry`/`\restoregeometry` is the correct approach for per-page layout changes
