### What was built
A 5-page test document (`sudodoc-elements.tex`) showcasing all document elements:

**Code Listings (Section 3):**
- Syntax highlighting via `listings` package: blue keywords, green strings, gray comments
- Left accent bar (2pt, codeKeyword color), light codeBg background
- Line numbers, monospaced font, proper caption style

**Blockquotes (Section 4):**
- tcolorbox-styled with warm quoteBg background, left blue accent bar (3pt)
- Opening/closing quotation marks rendered with TikZ
- Italic text for distinction
- VLM rated **8/10** — "subtle, cohesive palette"

**Figures & Math (Section 5):**
- TikZ-drawn placeholder diagram (Raft cluster topology) with nodes and arrows
- Caption with "Figure 1" label
- Display math with `amsmath` — binomial probability formula
- Description list for CAP theorem terms
- VLM rated **8/10** — "polished, academic-style"

### Key technical challenges
1. **TikZ-enumitem conflict**: TikZ semicolons get redefined inside enumitem list contexts. Fixed by using math symbols ($\blacktriangleright$, etc.) instead of TikZ for list markers.
2. **Package load order**: `enumitem` must be loaded BEFORE `titlesec` to avoid `\insertsubsection` becoming undefined.
3. **\captionsetup with \color**: Using `\color{}` inside `font={...}` keys of captionsetup caused `\@argdef` scan errors. Fixed with simple `font=small, labelfont=bf` setup.
4. **`%` in \renewcommand**: A `%` after `{` in `\renewcommand{cmd}{%content}` can cause brace-scanning issues during TeX's low-level `\def` processing. Avoid `%` on the opening brace line.

### Lessons
- LuaLaTeX compilation with many TikZ elements can take 3-5 minutes on 4 vCPU — use `nohup` or background compilation
- Math symbols ($\blacktriangleright$, $\blacksquare$, $\square$) work as drop-in replacements for TikZ bullets
- Always check brace balance with a script before compiling complex LaTeX files
